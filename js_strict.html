JS Strict

  • the "use strict" directive
  • literal expression
  • ignored by older versions of JS
  • purpose: code should be executed in a strict manner
  • example: cannot use undeclared variables
    Declaring strict mode
  • add "use strict" in the beginning of a js file or js function
  • if used in top of js file it has global scope
  • if used in a function it has local scope
    global declaration

    local declaration
  • x = 3.16; // no error will happen notice var is missing
    afunc(); // error will happen
    function afunc() { "use strict" ; x = 3.16;}
    "use strict" syntax
  • the declaration of strict mode was designed to be compatible with older versions of JS
  • compiling literal like (1+2;) or a string literal like ("Betsy Blake";) in a JS program has no side effects
  • it will compile to a none existent variable and poof
  • the strict mode only matters to a new compiler that knows what to do
    why use strict mode?
  • you can write more secure JS
  • changes bad syntax that could work into actual errors
  • without strict mode you could accidentally create a global variable by mistyping a variable name
  • in strict mode this will throw an error instead of creating a global variable
  • in JS normally you would not receive error feedback assigning values to non-writable properties
  • strict mode: assignments to non-writable properites will throw an error
  • examples: non-writable property, getter-only property, non-existing property, non-existing variable, non-existing object all will throw errors
    things not allowed in strict mode
  • "use strict"
  • x = 3.16; // this will cause an error as x has not been declared
  • delete x; // cannot delete a variable, function, or argument
  • var y = {p1:3.14, p1:3:16}; // cannot define same property more than once
  • function myFunc(arg1, arg1){}; // duplicating parameters
  • var x = 010; // no octal numeric literals
  • var y = \010; // no escape characters
  • var myObject = {}; // empty object
  • Object.defineProperty(obj, property, decriptor)
  • Object.defineProperty(myObject, "x", {value:0, writable:false});
  • myObject.x // returns zero
  • myObject.x == "writable"; false
  • myObect.x = somethingElse; // errors out
  • var myObject = {getx() {return 0} }; // get-only property
  • myObject.x = 3.16; // error
  • delete Object.prototype; // error ; cannot delete undeletable property not allowed
  • var eval = 3.14; // error ; cannot use eval as a variable
  • var arguments = 3.16; // error; cannot use arguments as a variable
  • with (Math){x = cos(0)}; // with statement not allowed
  • using eval to create variables and calling within the same scope not allowed
  • eval ("var d = 1337"); alert(d); // error
