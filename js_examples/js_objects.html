JS objects

    JS objects review
  • Arrays are always objects
  • other languages have associative arrays, but for JS see Objects
  • Booleans can be objects / primitives that act like objects
  • Dates are always objects
  • Functions are always objects
  • Maths are always objects
  • Numbers can be objects / primitives that act like objects
  • Objects are objects
  • Regular expressions are always objects
  • Strings can be objects / primitives that act like objects
  • Basically all values are objects except primitive values
    Objects and variables
  • Objects are really just variables used to contain variables
  • variables generally contain 1 value
  • example 1:
    var car = "ferrari";
    var car_object = {make:"toyota", model:"tundra", color:"lightning"};
  • example 2 is written in name:value pairs
  • JS objects basically are just containers that house named value pairs
  • These named value pairs are referred to as properties
  • property:property value
  • firstName:"john"
  • lastName:"matthew"
    object methods
  • You can store methods in the property value
  • You will make function and store it as a property value
  • fullName:function(){return this.firstName + " " + this.lastName;}
    object definition
  • When you define and create objects you use object literals
  • var person = {
    firstName:"juan",
    lastName:"bautista",
    age:33,
    beard:"yes"
    };
    creating JS objects
  • You may define and create your very own objects
  • 3 basic ways to create objects
  • Use an object literal to both create and define an object
  • Use the new keyword to both create and define an object
  • 1) Create an object constructor definition. 2)use object construtor to create objects
  • With ECMAScript 5 you can just use this function: function Object.create()
    Getting to those object properties
  • example 1:
    object_name1.property_name1;
  • example 2:
    object_name2["property_name1"];

  • object_name1.method_name1()

    Object constructor
  • Want to create more than just one object and save time?
  • function vehicle(make, model, color, drive){
       this.make = make;
       this.model = model;
       this.color = color;
       this.drive = drive;
    }
  • var neighbor1_truck = new vehicle("toyota", "t100", "blue", "2wd");
  • var neighbor2_car = new vehicle("ford", "pinto", "red", "fwd");
    Using the this keyword
  • In JS there is an object that owns the JS code
  • That object is called this
  • When used in a function it's value is the object that owns the function
  • When used in an object it's value is the object itself
  • If you use it in an object constructor it lacks a value. It is a substitute for the new object
  • When the constructor is used this will become the new object
  • this is not a variable. It is a keyword.

    JS objects and mutability
  • In JS objects are mutable.
  • Immutability: Once the object is instantiated you cannot change it's properties
  • In lower levels immuntability will store a string in some memory allocated for it. If you modify the string it will store that in a different piece of memory. It essence it sticks around until memory is cleared; making strings expensive. Although this may have changed over the years
  • Primitive types are immutable: null, undefined, boolean, number, string, symbol
  • Immutable values are incapable of being changed in JavaScript ECMAScript 6:
  • Objects on the other hand are address by reference instead of value
  • example 1:
    var a = {value:"pair"};
    var b = a; // does not clone a the object
    both a and b point (reference) the to same object now
  • If you change a above it will also change because they both point to the same object
  • example 2:
    var vehicle = {make:"ferrari", model:"enzo", year:2007, color:"red"};
    var myCar = vehicle;
    myCar.color = "blood red"; // now both variables point to 'blood red';
  • Reference. Remember reference
