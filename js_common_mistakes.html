    Floats why are they so hard?
  • JS uses floating point numbers. 64bits
  • All programming languages have problems when it comes to floating point value precision
  • var a = 0.1; var b = 0.2; var c = a + b;
  • what will c be? 0.3? Nope it is 0.30000000000000004
  • to get around this basically go up to whole numbers then divide to drop it back down
  • var d = (a * 10 + b * 10) / 10; // d will be 0.3

    Accessing Arrays with Named indexes
  • Named indexes are associate arrays or hashes (Ruby)
  • JS does NOT support associated arrays
  • JS supports numbered indeces in arrays
  • var thisArray = ["john", "paul", "matthew"];
  • thisArray.length; // returns 3
  • thisArray[1]; // returns "paul"
  • if you want to name your indexes use objects
  • var person = [];
  • person["firstName"] = "john";
  • person["lastName"] = "silver";
  • person[0]; // undefined
  • person["firstName"]; // returns "john"
  • person.length; // returns 0
  • person.lastName; // returns silver
