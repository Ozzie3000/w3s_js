JS function invocation

    A bit of info about invocation
  • 4 ways to do so
  • Each method will vary on how "this" is initialized
    Invoking a JS function
  • Remember the function (stuff inside) executes when it is called/invoked
  • Think of invoking and calling as being the same
  • Defining a function does not invoke it
  • In fact some say: call, invoke, start
    Invoking a function as a function
  • Remember the code block inside a JS function executes when something calls it
  • function myFunc(a, b){
      return a * b;
    }
  • myFunc(5, 6); // returns 30
  • myFunc does not belong to an object, however in JS there is always a default global object
  • In a browser the page object is the browser window
  • So the function will become a window function
  • myFunc() == window.myFunc()
  • Think of alert, location, etc
  • alert() == window.alert()
  • Don't use these type of calls though; think of conflicts and all
    The Global object
  • Invoke a function without an object owner and itwill return the global object when you use this
  • (function gogo() { return this; })()
  • returns the window. probably http://www.somethinghere.com/blah/blah
  • Using the window object as a variable can crash stuff
    Invoking a function as a method
  • In JS you can define functions as object methods
  • var anObject = {
      firstName: "john", // first property
      lastName: "matthew", // second property
    fullName: function(){ //function as a method
        return this.firstName + " " + this.lastName;
      } }
  • anObject.fullName(); // returns "john matthew"
  • fullName is a method which is a function
  • The function belongs to the object known as anObject
  • this owns the JS code so for this example the value of this is anObject
  • var myObject = {
      firstName:"John",
      lastName: "Doe",
      fullName: function () {
        alert(this + ":using alert for this");
        console.log(this + ":using console.log for this");
        //"document.write(this);
      }
    }
  • myObject.fullName();
  • console.log == Object { firstName: "John", lastName: "Doe", fullName: myObject.fullName() }
  • alert == [object Object]
  • document.write == [object Object]
  • innerHTML == [object Object]
  • Invoking a function as an object method will make the value of this be the object itself
    Invoking a function with a function constructor
  • Use the keyword new to make a function; you've got a constructor invocation
  • *** Function constructor ***
  • function myFunc(argument1, argument2){ this.firstName = argument1; this.lastName = argument2; }
  • ***Creating a new object ***
  • var object1 = new myFunc("jon", "snow");
  • object1.firstName; // returns "jon"
  • A constructor invocation create a new object. That object inherits the properties and methods from the constructor used to make it
  • The keyword this in the constructor does not have a value
  • If you use alert(this) in there it will say [object Window]
  • The value of this will be the new object created when the function is invoked
    Invoking a function with a funcition method
  • Remember in JS function are object and they have methods and properties
  • call() and apply() are baked-in JS function methods
  • Both can be used to invoke a function and both methods must have the owner object as first parameter
  • function myFunc(a, b){ return a * b; }
  • anObject = myFunc.call(anObject, 5, 6); // returns 30
