JS hoisting: is JS's defaul behavior or moving declarations to the top

    JS declarations are hoisted
  • a variable can be declared after it has been used
  • therefore a variable can be used before it has been declared
  • example 1
  • x = 1; assignment (also notice keyword var is missing)
  • alert(x); // windows shows 1
  • var x; // declaration
  • ------
  • example 2
  • var y; // declaration
  • y = 7; // assignment (also notice keyword var is missing)
  • alert(y); // window shows 7
    JS initializations are not hoisted
  • var a = 1; // initialize a (also notice keyword var is here)
  • var b = 2; // initialize b (also notice keyword var is here)
  • console.log(a + " " + b);
  • vs
  • var c = 3; // initiallize c
  • alert(c + " " + d);
  • var d = 4; // initialize d
  • it will say (3 undefined)
  • only the declaration (var d), not the initialization (=4) gets hoisted to the top
  • hoisting moves the declaration to the top before the assigment/ initialization is made making the variable undefined
