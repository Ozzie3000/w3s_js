    JavaScript Numbers
  • JS only has 1 type of number
  • JavaScript Numbers are Always 64-bit Floating Point
  • JavaScript numbers are always stored as double precision floating point numbers, following the international IEEE 754 standard.
  • Integers (numbers without a period or exponent notation) are considered accurate up to 15 digits.
    examples:
    var a = 999999999999999; // a will be 999999999999999
    var b = 9999999999999999; // b will be 10000000000000000
  • The maximum number of decimals is 17, but floating point arithmetic is not always 100% accurate:
    var x = 0.2 + 0.1; // x will be 0.30000000000000004
  • to over come this multiple and then divide
    var y = (0.2 * 10 + 0.1 * 10)/10 // y will be 0.3
    • hexadecimal
    • JS interprets numeric constants as hexadecimals if it starts with 0x
    • var a = 0xBB; // a will be 187
    • use the toString() method to output numbers as base 16 (hex), base 8 (octal), or base 2 (binary).
    • var aNumber = 128;
    • aNumber.toString(16); // returns 80
    • aNumber.toString(8); // returns 200
    • aNumber.toString(2); // returns 10000000

    • NaN
    • NaN is a JavaScript reserved word indicating that a value is not a number.
    • var a = 10/"apple"; // a will be NaN
    • or you can use the global js function isNaN()
      isNaN(a); // returns true
    • var b = 100/"10"; // b will be 10. The JS interpreter converts "10" to 10.
    • if you use NaN in a math operation it will return NaN
      var a = 10; var b = NaN; var c = a + b; c will be NaN
