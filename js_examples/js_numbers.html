JS numbers

    The one and only
  • JS only one type of numbers
  • Think JAVA: float, double, etc
  • In JS stored as double precision floating point numbers
  • IEEE 754 international standard
  • It is a 64 bit storage
    Storage breakdown
  • numbers aka fraction gets stored in bits 0 through 51
  • exponents get stored in 52 through 62
  • sign gets stored in bit 63
    Accuracy (most call it precision)
  • numbers only using storage 0-51 are accurate up to 15 digits
  • examples:
    var a = 99999999999999; // same
    var b = 999999999999999; // changes to 10000000000000000
  • maximum precision is 17 for decimals
  • var x = 0.01 + 0.02; // 0.30000000000000004
  • var x = 0.1 + 0.2 + 0.3; // 0.6000000000000001
  • work arounds:
    use whole numbers then divide at end
    var y = (0.1 * 10 + 0.2 * 10 + 0.3 * 10) / 10; // 0.6
    Hexadecimal
  • preciding 0x
  • var a = 0xAB; // 171
  • careful not to write numbers with a 0 up front
  • 01 for example. Some JS version treat this as octal
  • by default JS stores numbers as base 10 decimals
  • but there is nothing stopping you from converting
  • var example1 = 256;
  • var a = myNumber.toString(16); // a == 100
  • var b = myNumber.toString(8); // b == 400
  • var c = myNumber.toString(2); // c == 100000000
    NaN
  • Not a number
  • var silliness = 50 / "bears";
  • silliness will be NaN
  • var laziness = 50 / "50";
  • laziness == 1;
  • JS has a global function to check for this called isNaN();
  • var silliness2 = 10 / "turkey";
  • isNaN(silliness2); // returns true
  • var a = NaN;
  • var b = 2;
  • var c = a + b; // c is still NaN
