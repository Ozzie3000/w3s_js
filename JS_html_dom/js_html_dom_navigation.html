JS HTML DOM navigation

    Tree, nodes, elements
  • THe entire tree is a document node
  • HTML elements are element nodes
  • Text inside those HTML elements are text nodes
  • HTML attributes are attribute nodes
  • Comments are [you guessed it] comment nodes
  • You can use JS to access all those nodes
  • You can also create nodes
  • You can modify nodes
  • You can delete nodes
    Node relationships
  • Nodes have hierarchical relationships to one another
  • Words used to decribe relationships: parent, child, sibling
  • The top most node is the root (a tree up-side down I suppose)
  • Every node has 1 parent (with the exception of root)
  • A node can have various children
  • Siblings share a common parent

  • body is the last child
  • head and body are siblings
  • head has one child which is the title
  • title has one child the text node: node examples
  • body has two children: h1 and p
  • h1 and p are sibilings
  • h1 has one child: OM... DOM, DOM... DOM
  • p has one child: "Weh-woh"

    The nodeName property
  • nodeName:
  • specifies the name fo the node
  • is read-only
  • of an element is = tag name
  • document.getElementById("example1").nodeName; // returns "LI
  • document.getElementById("example1").nodeName[1]; // returns "I"
  • of an attribute node is = attribute name
  • of a text node is #text
  • of the document is #document
  • They also contain the HTML element's uppercase tag name
    nodeValue property
  • Specifies the value of a node
  • nodeValue for element nodes id undefined
  • nodeValue for text nodes is the text
  • nodeValue for attribute nodes is the value of the attribute
    nodeType property
  • nodeType property returns the type of node
  • nodType is also read-only
