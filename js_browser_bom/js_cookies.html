JS cookies

    Cookies
  • Store info in text files about websites
  • Cookies were created to help servers remember browsers
  • They help solve the problem of continuity
  • When a user hits a page inof in stored in the cookie about the user
  • Next visit (assuming cookie hasn't been deleted or expired) the web page will read it and respond
  • They store info in name-value pairs
  • Web a browser requests a page from a server it also send info about the cookie if it belongs to the same domain
    Making cookies with JS
  • JS can create, delete, read cooies
  • ticket is document.cookie property
  • document.cookie="username=john luke";
  • document.cookie="username=John luke; expires=Tue, 10 Mar 2020 12:00:00 UTC";
  • document.cookie="username=john luke; expires=Tue 10 Mar 2020 12:00:00 UTC; path=/";
  • This tells the browser what path the cookie should belong to.
  • By default it belogs to the current page
