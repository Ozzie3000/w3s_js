JS performance

    loops
  • Place searching for statements or assignments outside loops
  • for (x = 0; x < myWhatever.lenghth; x++){ code block} // newbs
  • size = myWhatever.length;
    for (x = 0; x < size; x++){ code block }
  • First one access length of what you are searching through
  • Second sets length outside of loop and does not check length everytime it loops
    Reduce accessing the DOM
  • Accessing the DOM is slow compared to other JS activities
  • If you need to access it multiple times try storing info in local variables
    Avoid bloating. Reduce variables
  • Creating and accessing variables that are used very little to none is a waste of time
  • ex: fullName = firstName + " " + lastName;
  • document.getElementById("sample").innerHTML = fullName;
  • ex 2: document.getElementById("sample").innerHTML = firstName + " " + lastName;
  • skip the creation of a variable you will only use once
    Delay JS loading
  • Putting Scripts at the bottom of the page lets the rest of them DOM load
  • If a script is downloading, the browser will not start any other downloads.
  • As a result other rendering and parsing might be held up
