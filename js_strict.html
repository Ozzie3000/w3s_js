JS Strict

  • the "use strict" directive
  • literal expression
  • ignored by older versions of JS
  • purpose: code should be executed in a strict manner
  • example: cannot use undeclared variables
    Declaring strict mode
  • add "use strict" in the beginning of a js file or js function
  • if used in top of js file it has global scope
  • if used in a function it has local scope
    global declaration

    "use strict" syntax
  • the declaration of strict mode was designed to be compatible with older versions of JS
  • compiling literal like (1+2;) or a string literal like ("Betsy Blake";) in a JS program has no side effects
  • it will compile to a none existent variable and poof
  • the strict mode only matters to a new compiler that knows what to do
    why use strict mode?
  • you can write more secure JS
  • changes bad syntax that could work into actual errors
  • without strict mode you could accidentally create a global variable by mistyping a variable name
  • in strict mode this will throw an error instead of creating a global variable
  • in JS normally you would not receive error feedback assigning values to non-writable properties
