JS function definitions

    Function declarations
  • function functionName(params) { code block; }
  • they are stored to memory until invoked
  • ex: onload="aFunc" or a self-invoking function
    Function expressions
  • JS function is also known (defined as) as an expression
  • A function expression can be stored in a variable
  • var multiplier = function(a,b){ return a * b };
  • Crazy fun starts. Now you can use the variable as a function
  • console.log(multiplier(5 * 5)); // returns 25 in console
  • Info nuggets
  • multiplier is storing an anonymous function
  • Functions stored in variables do not need for function to have a name. They are invoked when using the variable
    Function() constructor
  • Previous examples have shown us the function keyword is used to define JS functions
  • You may also use Function(); a baked-in function constructor
  • var myFunc = new Function("a", "b", "return a * b");
  • console.log(myFunc(5,6)); // logs 30
  • kind of like this one below:
  • var myFunc2 = function(a,b) { return a * b};
  • console.log(myFunc2(7,8));
    Function hoisting
  • Recap: JS's default behavior to move (hoist) declarations to the top of current scope
  • Applies to both variable and function declarations
  • This allows JS functions to be invoked before they are declared
  • myFunc(13)
  • function myFunc(x){ return x * x; };
  • Watch out! Functions definted using an expression are not hoisted
    Self-Invoking functions
  • Function expressions can be self-invoking
  • Function invoked automatically without being called
  • This is when a function is executed automatically when appended with ()
  • Cannot self-invoke a function declaration
  • (function(){ alert("hi"); })();
  • (function(){
      console.log("fail");
    })();
  • both are self-invoking anonymous functions
    Functions as values
  • function myFunction(a, b){ return a * b; }
  • var x = myFunction(5, 6);
    Function are objects
  • Use typeof on a JS function it will tell you it is a "function"
  • Lets think of them as objects instead
  • JS function have both methods and properties
  • arguments.length property tells us how many arguments were receive when the function was invoked
