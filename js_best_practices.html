JS Best practices

    common things to avoid
  • Global variables. Known as poluting the global namespace
  • using new
  • using == // looking for strict in type and value may save you problems
  • using eval()
    The Global variable
  • Avoid this when possible
  • Including all data types, objects, and funcions
  • Global variables and functions can be overwritten by other scripts
  • Preferrably use local variables and closures.
  • Basically using anonymous functions to wrap your function and protect variables
    Local variables
  • Always declare local variables
    Declarations at the top
  • Gives you cleaner code; readability
  • Single spot to look for local variables
  • Easier to troubleshoot. Ex implied global variables
  • Reduce conflicts; re-declarations
    Initialize variables
  • be a good coder. Intialize variables when you declare them if you can
  • cleaner code
  • use a single location to intialize variables
  • stay away from undefined values
  • var firstName = "",
    lastName = "",
    birthPlace = "",
    price = 0,
    thatArray = [],
    someObject ={};
  • this will give you an idea for future use and data type
    Number, String, or Boolean objects should ot be declared
  • treat those three as primitive values
    what to use instead
  • instead of ... use ...
  • new Object() // use {}
  • new String() // use ""
  • new Number() // use 0
  • new Boolean() // use false
  • new Array() // []
  • RegExp() // /()/
  • new Function() // function(){}
  • ------------------------------
  • how they would look
  • var a = {}; // new object
  • var b = ""; // new primitve sring
  • var c = 0; // new primitive number
  • var d = false; // new primitive boolean
  • var e = []; // new array object
  • var f = /()/; // new regexp object
  • var g = function(){}; // new function object
    Automatic type conversions
  • since JS is a weakly / loosely typed language things can happen
  • var stuff1 = "howdy"/ // typeof stuff1 is a string
  • var stuff1 = 69; // now typeof stuff1 is a number
  • var stuff2 = 123 + "5"; // stuff2 == "1235" so typeof stuff2 is string
  • var stuff3 = 2 - "5"; // stuff == -3 fo typeof stuff3 is a number
  • Var string1 = "hello"; // var string2 = "world";
  • "hello" - "world" // returns NaN
    comparisons
  • == converts to matching types before it runs the comparison
  • === checks value and type
  • 0 == ""; // true
  • 1 == "1" // true
  • 1 == true // true
  • 0 === ""; // false
  • 1 === "1"; // false
  • 1 === true; // false
    Parameter defaults
  • if a function is called, but one of the arguments is missing it will make that argument undefined
  • this can break your code
  • this is why you should declare and initialize with default values
  • function myFunc(a,b) { if (b === undefined){ b = 0; } }
