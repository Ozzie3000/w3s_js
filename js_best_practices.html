JS Best practices

    common things to avoid
  • Global variables. Known as poluting the global namespace
  • using new
  • using == // looking for strict in type and value may save you problems
  • using eval()
    The Global variable
  • Avoid this when possible
  • Including all data types, objects, and funcions
  • Global variables and functions can be overwritten by other scripts
  • Preferrably use local variables and closures.
  • Basically using anonymous functions to wrap your function and protect variables
    Local variables
  • Always declare local variables
    Declarations at the top
  • Gives you cleaner code; readability
  • Single spot to look for local variables
  • Easier to troubleshoot. Ex implied global variables
  • Reduce conflicts; re-declarations
    Initialize variables
  • be a good coder. Intialize variables when you declare them if you can
  • cleaner code
  • use a single location to intialize variables
  • stay away from undefined values
  • var firstName = "",
    lastName = "",
    birthPlace = "",
    price = 0,
    thatArray = [],
    someObject ={};
  • this will give you an idea for future use and data type
    Number, String, or Boolean objects should ot be declared
  • treat those three as primitive values
