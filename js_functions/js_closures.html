JS closures

    fun facts
  • JS variables can have global or local scope
  • Private variables can be made possible with closures
    Global variables
  • Reminder: a function has access to all variables defined inside of it
  • example:
  • function myFunc(){ var a = 4; return a * a; }
  • Variables defined outside can be accessed as well
  • var b = 5;
  • funciton myFunc2(){ return b * b; }
  • var a has local scopre
  • var b has global scope
  • on a browser var b belongs to the window object
  • global variables can be accessed and changed by all scripts in the page and window. [for better or worse]
  • Trickery: Global variables can have the same name
  • Changing one or the other will not modify the other
    Variable lifetime
  • Global variables live as long as the [window, application, web page, w/e] lives
  • Local variables are transient, ephemeral, fleeting, short-lived
  • Local variables are created when the function is invoked
  • Local variables are destroyed when the function is finished
    Counting can be so complicated
  • Counting is everywhere: failed log in attempts, etc
  • Global variable example
  • var counter = 0;
  • function addUp() { counter += 1; }
  • addUp(); // now 1
  • addUp(); // now 2
  • addUp(); // now 3
  • Downside is you can access counter and change it without calling the add() function. It is a global var
  • Local variable example:
  • function addUp2(){ counter = 0; counter += 1; }
  • addUp2(); // line 1 it gets set to 0 and then line two it gets incremented by 1
  • addUp2(): // line 1 it gets set to 0 and then line two it gets incremented by 1
  • addUp2(): // line 1 it gets set to 0 and then line two it gets incremented by 1
  • that first line keeps resetting it
