    Assignment operator
  • var a = 3;
  • if (a == 10)
  • a case unexpected input and values Boolean(x = 10); // Boolean will evaluate that to true (sets x to 10) unexpected
  • a case unexpected input and values Boolean(x == 10); // Boolean will evaluate that to false (checks as you expect)
  • a case unexpected input and values Boolean(x = 0); // Boolean will evaluate that to false (sets x to 0 // which converts to false) unexpected
    Expecting loose comparison
  • == data type does not matter if statement returns true
  • switch cases use strict comparison
  • "1" === 1 ; will not work
    Addition and concatenation confusion
  • We all know addtion // 2+2
  • Concatenation is to join two strings together
  • Both use the same operand the plus symbol
  • Addtion: 2 + 2 // equals 4
  • Concatenation 2 + "2" // equals "22"
  • The concatenation did a type converation for the number 2 into a string
    Floats why are they so hard?
  • JS uses floating point numbers. 64bits
  • All programming languages have problems when it comes to floating point value precision
  • var a = 0.1; var b = 0.2; var c = a + b;
  • what will c be? 0.3? Nope it is 0.30000000000000004
  • to get around this basically go up to whole numbers then divide to drop it back down
  • var d = (a * 10 + b * 10) / 10; // d will be 0.3
