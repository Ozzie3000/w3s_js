JS object properties: the most important part of JS object

    JS properties
  • the values associated with the JS object
  • JS object is an unordered collection of properties
  • properties can be added, changed, deleted
  • some properties are read-only
    Accessing JS object properties
  • objectName.property // i.e. personHood.age
  • objectName["property"] // i.e. personHood["age"]
  • objectName[expression] // a = "something"; personHood[a]
    JS for ... in loop
  • for (variable in object){
      code block to be executed
    }
  • the code block gets evaluated once for each loop for each property
  • var personHood = {firstName: "john", lastName: "steinbeck", age: 40};
  • for (x in personHood){
      stuff += personHood[x] + " ";
    }
  •  
    Adding new properties
  • simply add properties by giving values
  • the object must already be in existence
  • we will use the personHood
  • cannot use keywords or reserved words for methods or properties
  • personHood.nationality = "U.S.";
  • call it : personHood.nationality; // returns "U.S."
    Deleting properties
  • The keyword delete makes the magic happen
  • var personHood = {firstName: "john", lastName: "steinbeck", age: 40, nationality: "U.S."}; //in case I forget
  • lets delete a single property and it's value
  • delete personHood.age;
  • or
  • delete personHood["age"];
  • in console it will return true (it will also return true if you make stuff up // delete personHood.blahblahblah; // true)
  • want it back? add it back in too use it again
  • delete operator has no effects on variables or functions
  • do not use it on predefined JS object properites
  • if you do it probably will break stuff
    Property Attributes
  • Properties have names and values
  • example of attributes:
  • value of a property
  • enumerable
  • configurable
  • writable
  • these all define how the property can be accessed
  • in JS all attributes can be read
  • only the value attribute will let you change it (also it has to be writable)
  • ECMAScript 5 has methods for setting and getting all property attributes
    Prototype properties
  • inheritence; oh boy
  • JS objects inherit properties of their prototype
