JS errors

    JS errors throwing and catching
  • errors will happen
  • errors can be a result of programming, bad input, unexpected behavior
  • try statement lets you test a block of code to look for errors
  • catch statement lets you handle errors
  • throw statement let your create custom errors
  • finally statement lets you execute code, after try and catch, in any result
    try and catch example
  • first error
  • the try section of the code has a typo
  • the catch code block see the error and executes a code block to handle it
  • try { the try code block goes here } catch(error) { code block that handles errors };
  • try - statement lets you define a code block to be tested for errors while it is being executed
  • catch - statement lets you define a code block to be executed if an error it caught in the try code block
    throw errors
  • normally JS stops when an error occurs and generate a message
  • this is known as JS throwing an error
    throw statements
  • throw statement lets you create a custom error
  • known as: throw an exception
  • the exception can be a JS string, Number, or Object
  • throw "this right here"; // throw a string
  • throw 123; // throw a number
  • combine try, catch, and throw and now you can create custom errors
