    Strings can be Objects
  • Generally you refer to strings as being primitive values
  • You use literals or string literals to make them
  • var car1 = "viper";
  • You may use the new keyword to defined them as objects
  • var car2 = new Strign("viper");
  • typeof car1; // string
  • typeof car2; // object
  • Things will get messy
  • car1 == car2; // will say true because loosely they contain same values
  • car1 === car2; // false do not match in type
  • Also frowned upon making strings objects
  • Objects are unique so even if you make two object contain same values they are not the same
    String methods and properties
  • Objects have methods and properties
  • primitive values do not have methods and properties
  • JS does a slight of hand, and will treat strings as objects so you can execute some methods and properties
